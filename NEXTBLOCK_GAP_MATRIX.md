## Note di baseline

1. **Legacy mappato, non toccato:** la dir `src/` alla root è un'app Vite residua (con propria integrazione Supabase) precedente alla migrazione Next in `app/`. Va rimossa/archiviata in uno step dedicato, non in questo.
2. **Single-EOA staging:** lo staging 84532 è reale ma interamente operato dal deployer `0xfF6f…81d2` su tutti i ruoli. È accettabile per demo, è il gap più critico verso lo standard (riga Governance).
3. **`Keeper.s.sol` esiste ma è ritirato:** il file è presente come stub documentale del ritiro Phase 9.5; non c'è alcun keeper operativo.
4. **Foundry non presente sulla macchina di snapshot:** i comandi `forge`/`cast` di questa matrice sono eseguibili solo in ambiente con Foundry installato.
5. **Nessuna riga di questa matrice è "done":** anche le aree con stato avanzato (contracts, frontend Lens) restano aperte finché il rispettivo criterio di completamento binario non è verificato.
6. **CI attiva e verde su GitHub Actions (aggiornamento 2026-06-11):** repository pubblicato su `https://github.com/antoncarlo/nextblock` (main), pipeline a tre job (`addressbook`, `frontend`, `contracts`) interamente verde dalla run 27314071700. Dipendenze Foundry come submodule pinnati (forge-std v1.9.7, openzeppelin-contracts v5.4.0). Foundry e' ora installato anche in locale (forge 1.5.1): l'intera suite contracts (357 test) e' verificabile prima del push. Verifica locale: script `npm run ci:frontend:*`, `npm run ci:contracts:*` (con `~/.foundry/bin` nel PATH) e `npm run check:addressbook` alla root.
